CATATAN REVISI - API LUAS AREA KERAWANAN
=========================================

Perubahan utama:
1. Backend server.js ditambahkan endpoint:
   /api/area-stats?layer=rf_all
   /api/area-stats?layer=slope_class
   /api/area-stats?layer=rain_class
   /api/area-stats?layer=landcover_class
   /api/area-stats?layer=rf_slope_rain
   /api/area-stats?layer=rf_slope_landcover
   /api/area-stats?layer=rf_rain_landcover

2. Endpoint /api/area-stats menghitung luas aman, sedang, rawan dari Google Earth Engine menggunakan pixelArea.

3. Mapping kelas sudah dibedakan:
   - Parameter dasar: 1=Aman, 2=Sedang, 3=Rawan/Tinggi
   - Random Forest: 0=Aman, 1=Sedang, 2=Rawan/Tinggi

4. dashboard.js sudah dihubungkan ke backend API sehingga card Statistik Kerawanan menampilkan:
   - Persentase aman, sedang, rawan
   - Luas aman, sedang, rawan dalam hektar
   - Total area analisis dalam hektar

5. dashboard.php sudah ditambahkan elemen:
   - statSafeArea
   - statMediumArea
   - statHighArea

6. dashboard-redesign.css sudah ditambahkan styling agar teks hektar tampil rapi di bawah persentase.

7. Informasi Lokasi dibuat default jika tidak ada parameter/RF yang dipilih:
   - Lat: - , Lon: -
   - Status: -
   - Slope: -
   - Rain: -
   - Landcover: -
   - Score: -

8. Checkbox RF semua parameter pada dashboard tidak lagi aktif otomatis dari HTML, sehingga tampilan awal bisa kosong/default.

Cara menjalankan:
1. Buka terminal di folder proyek.
2. Jalankan:
   npm start
3. Uji backend:
   http://localhost:3000/api/status
   http://localhost:3000/api/area-stats?layer=rf_all
4. Buka dashboard.php melalui server lokal PHP/XAMPP.
5. Pilih parameter/layer, card Statistik Kerawanan akan mengambil data luas dari backend.

Catatan:
- Pastikan komputer terhubung internet saat memakai endpoint Google Earth Engine.
- Pastikan service-account-key.json valid.
- Pastikan semua asset GEE sudah di-share ke email service account sebagai Reader.
- Jika GEE_AREA_STUDI_ASSET bermasalah, cek nilai di file .env.
